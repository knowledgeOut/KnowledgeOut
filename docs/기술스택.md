# 기술 스택 문서

## 백엔드 (Backend)

### 프레임워크
- **Spring Boot 3.5.9**
  - 웹 애플리케이션 프레임워크
  - RESTful API 서버 구축

### 언어 및 런타임
- **Java 21**
  - 최신 LTS 버전 사용
  - Modern Java 기능 활용

### 빌드 도구
- **Gradle**
  - 의존성 관리 및 빌드 자동화
  - Spring Boot Gradle Plugin 사용

### 데이터베이스
- **MySQL 8.0**
  - 관계형 데이터베이스
  - Docker Compose를 통한 컨테이너 실행
  - UTF-8 인코딩 지원 (utf8mb4)

### ORM / 데이터 접근
- **Spring Data JPA**
  - 데이터 접근 추상화
  - Repository 패턴 구현
- **Hibernate**
  - JPA 구현체
  - DDL 자동 생성 (update 모드)

### 보안
- **Spring Security**
  - 인증(Authentication) 및 인가(Authorization)
  - 세션 기반 인증 (Session-based Authentication)
  - BCrypt 패스워드 암호화
  - CORS 설정
  - Role-based Access Control (ROLE_ADMIN, ROLE_USER)

### 유효성 검증
- **Spring Boot Starter Validation**
  - 입력 데이터 유효성 검증
  - Bean Validation API

### 개발 도구
- **Lombok**
  - Boilerplate 코드 감소
  - @Getter, @Setter, @RequiredArgsConstructor 등 활용

### 테스트
- **JUnit 5**
  - 단위 테스트 프레임워크
- **Spring Boot Test**
  - 통합 테스트 지원
  - Spring Security Test
  - Spring Data JPA Test

## 아키텍처 패턴

### 백엔드
- **레이어드 아키텍처**
  - Controller (웹 레이어)
  - Service (비즈니스 로직 레이어)
  - Repository (데이터 접근 레이어)
  - Domain/Entity (도메인 모델 레이어)
  - DTO (데이터 전송 객체)
- **RESTful API**
  - REST 원칙 준수
  - HTTP 메서드 활용 (GET, POST, PUT, DELETE)
- **Spring Security Filter Chain**
  - 요청 기반 인증/인가 처리

### 프론트엔드
- **Feature-based 구조**
  - 기능별 디렉토리 분리 (auth, question, answer, member, category, admin)
- **Component-based 구조**
  - 재사용 가능한 UI 컴포넌트
  - 공통 컴포넌트 분리 (common/)
- **API 클라이언트 추상화**
  - features 폴더 내 API 함수 정의
  - lib/axios.js를 통한 통합 API 클라이언트
